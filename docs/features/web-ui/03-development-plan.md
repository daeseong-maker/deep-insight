
## Development Plan — deep-insight-web

Based on architecture decisions in 01-plan.md and PoC results in 02-technical-proof-points.md.

## Features

### Feature 1: Health check — `GET /health` ✅ DONE

Simple JSON response for ALB health check. Returns status, runtime ARN, region, S3 bucket.

### Feature 2: Static page serving — `GET /` ✅ DONE

Serve `static/index.html` directly via `HTMLResponse`.

### Feature 3: File upload — `POST /upload` ✅ DONE

- Accept multipart form: data file (required) + column_definitions.json (optional)
- Supported data formats: CSV, Excel (.xlsx/.xls), TSV, TXT, JSON
- Generate an `upload_id` (UUID)
- Upload to S3 (keep original filename):
  - `s3://{bucket}/uploads/{upload_id}/{original_filename}`
  - `s3://{bucket}/uploads/{upload_id}/column_definitions.json` (if provided)
- Return `upload_id` + S3 paths in response

> **Note**: `upload_id` is generated by the web server for organizing uploaded files.
> This is distinct from the AgentCore `session_id` which is created by the runtime at analysis start
> and used for results under `deep-insight/fargate_sessions/{session_id}/`.

### Feature 4: Analysis + SSE streaming — `POST /analyze` ✅ DONE

- Accept JSON: `{ "upload_id": "...", "query": "..." }`
  - `upload_id`: returned from `/upload`
  - `query`: user's analysis prompt entered in the browser (e.g., "Analyze sales trends")
- Build payload: `{ "prompt": query, "data_directory": "s3://{bucket}/uploads/{upload_id}/" }`
- Call `boto3.invoke_agent_runtime()` with extended timeouts (3600s read)
- Parse SSE bytes from AgentCore → re-emit as SSE to browser via `StreamingResponse`
- Capture `session_id` from AgentCore SSE stream (needed for report download)
- Event types relayed: `agent_text_stream`, `agent_reasoning_stream`, `plan_review_request`, `plan_review_keepalive`, `workflow_complete`, `error`

### Feature 5: HITL plan review — `POST /feedback` + UI modal ✅ DONE

- During SSE streaming, server detects `plan_review_request` event and relays to browser
- Browser shows a modal with: plan text, revision count (e.g., "1/10"), timeout countdown
- User clicks **Approve** or types feedback + clicks **Reject**
- Browser sends `POST /feedback` with `{ "request_id": "...", "approved": bool, "feedback": "..." }`
- Server uploads JSON to `s3://{bucket}/deep-insight/feedback/{request_id}.json`
- AgentCore polls S3, reads feedback, continues or revises
- `plan_review_keepalive` events update the countdown timer in the modal
- Auto-approve on timeout (300s) is handled by AgentCore — no server-side logic needed

### Feature 6: Report download — `GET /artifacts/{session_id}` + `GET /download/{session_id}/{filename}` ✅ DONE

Two endpoints:
- `GET /artifacts/{session_id}` — list all files in `s3://.../fargate_sessions/{session_id}/artifacts/`, return filenames array
- `GET /download/{session_id}/{filename}` — read file from S3, return as attachment (`Content-Disposition: attachment`)

Download triggers:
- **Primary**: `workflow_complete` event → browser calls `/artifacts/{session_id}` → renders download link for each file
- **Shortcut**: If `workflow_complete` SSE event includes filenames, browser can show download buttons immediately

### Feature 7: Containerization — `Dockerfile` ✅ DONE

- `python:3.12-slim`, port 8080
- Must build with `--platform linux/amd64` for Fargate (PoC 5 lesson: ARM build → `exec format error`)

### Feature 8: Deployment — `deploy.sh` ✅ DONE

- Single script handles all infrastructure: ECR, Docker build/push, Security Groups, ALB, IAM, ECS task definition + service
- All on existing cluster (`deep-insight-cluster-prod`)
- ALB SG with VPN CIDR restriction created automatically
- `add_alb_sg_rule.sh` available for adding additional CIDRs after deployment

### Environment Variables

These variables are defined in `managed-agentcore/.env` (see `.env.example` for reference).
For local development, `app.py` auto-loads them from that file. For ECS deployment, `deploy.sh` injects them into the task definition.

```
RUNTIME_ARN=    # AgentCore Runtime ARN
AWS_REGION=     # AWS region (default: us-west-2)
S3_BUCKET_NAME= # S3 bucket for uploads/downloads
```

### Local Development

Reuse the `managed-agentcore/` UV environment (no separate venv needed):

```bash
# Install web dependencies (one-time)
cd managed-agentcore
uv add fastapi uvicorn python-multipart

# Run locally
cd managed-agentcore
uv run python ../under-development/front-end-implementation/deep-insight-web/app.py
```

`app.py` auto-loads env vars from `managed-agentcore/.env`. The server starts at `http://localhost:8080`.

## Infrastructure Decisions

### Reuse Existing Resources

| Resource | Name / ID | Notes |
|----------|-----------|-------|
| ECS Cluster | `deep-insight-cluster-prod` | No new cluster — add web service to existing cluster |
| VPC | Same VPC as managed-agentcore deployment | Place web server in same network |
| Subnets | Existing public subnets | ALB (internet-facing) and ECS task use same subnets |

> **Note**: The web app uses its own **internet-facing ALB** (`deep-insight-web-alb`), not the internal ALB used by AgentCore. Access is restricted to the deployer's VPN CIDR via ALB Security Group rules.

### ALB Security Group — VPN CIDR Restriction

`deploy.sh` creates the ALB Security Group with inbound TCP 80 restricted to the deployer's VPN CIDR (configured in `deploy.sh`).

To add **additional** allowed CIDRs after deployment, use the standalone script:

```
bash add_alb_sg_rule.sh <ALB_SG_ID> <ALLOWED_CIDR> [REGION]

# Example:
bash add_alb_sg_rule.sh sg-0abc1234 "10.0.0.0/8" us-west-2
```

- Adds TCP 80 inbound from the given CIDR to the ALB SG
- Idempotent — skips if the rule already exists

### New Resources to Create

| Resource | Name | Description |
|----------|------|-------------|
| ECR Repository | `deep-insight-web` | Docker image store |

## Deployment Flow

### Architecture

```
                         VPN CIDR only
                              |
                              v
  +--------+    HTTP    +------------+    TCP 8080    +------------------+
  |        | ---------> |    ALB     | ------------->  | ECS Fargate      |
  | Browser|    :80     | (internet- |                | (deep-insight-   |
  |        | <--------- |  facing)   | <------------- |  web / FastAPI)  |
  +--------+    SSE     +------------+    Response    +------------------+
                                                        |           |
                                              boto3 SSE |           | boto3
                                           (streaming)  |           | (S3)
                                                        v           v
                                                +------------+ +--------+
                                                | AgentCore  | |   S3   |
                                                | Runtime    | | Bucket |
                                                | (Bedrock)  | +--------+
                                                +------------+  uploads/
                                                      |         feedback/
                                                      v         artifacts/
                                                +------------+
                                                |  Fargate   |
                                                |  (code     |
                                                |  executor) |
                                                +------------+
```

All steps are automated in `deploy.sh`:

```bash
bash deploy.sh
```

`deploy.sh` performs the following steps in order:

1. **ECR Repository** — create if not exists
2. **Docker Build + Push** — build amd64 image, push to ECR
3. **Security Groups** — ALB SG (VPN CIDR restricted) + ECS SG (ALB-only inbound)
4. **ALB + Target Group + Listener** — internet-facing ALB, idle timeout 3600s
5. **IAM Task Role** — S3 read/write + AgentCore invoke permissions
6. **CloudWatch Log Group** — `/ecs/deep-insight-web`
7. **ECS Task Definition** — Fargate (256 CPU, 512 MEM), env vars injected
8. **ECS Service** — create or update on existing cluster

After deployment, wait for ECS stability:

```bash
aws ecs wait services-stable \
  --cluster deep-insight-cluster-prod \
  --services deep-insight-web-service \
  --region us-west-2
```

To add additional VPN CIDRs after deployment:

```bash
bash add_alb_sg_rule.sh <ALB_SG_ID> <ALLOWED_CIDR> [REGION]
```

## IAM Policy (Task Role)

This policy is created by `deploy.sh` and attached to the `deep-insight-web-task-role`.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "S3Upload",
      "Effect": "Allow",
      "Action": ["s3:PutObject"],
      "Resource": "arn:aws:s3:::{bucket}/uploads/*"
    },
    {
      "Sid": "S3Feedback",
      "Effect": "Allow",
      "Action": ["s3:PutObject", "s3:GetObject"],
      "Resource": "arn:aws:s3:::{bucket}/deep-insight/feedback/*"
    },
    {
      "Sid": "S3ArtifactsList",
      "Effect": "Allow",
      "Action": ["s3:ListBucket"],
      "Resource": "arn:aws:s3:::{bucket}",
      "Condition": {
        "StringLike": {
          "s3:prefix": "deep-insight/fargate_sessions/*/artifacts/*"
        }
      }
    },
    {
      "Sid": "S3ArtifactsGet",
      "Effect": "Allow",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::{bucket}/deep-insight/fargate_sessions/*/artifacts/*"
    },
    {
      "Sid": "AgentCoreInvoke",
      "Effect": "Allow",
      "Action": ["bedrock-agentcore:InvokeAgentRuntime"],
      "Resource": "*"
    }
  ]
}
```
