# Improve Chart Visibility

Initiative to fix chart legibility regressions in the DOCX reports
generated by Deep Insight (managed-agentcore + self-hosted).

## Goal

Make every chart in the final DOCX report readable on first glance —
clear axis labels, no clipped text, no overlapping annotations, dark
bold tick labels, and crisp vector embedding instead of bitmap-only PNGs.

## Approach

Pure prompt-engineering changes — no code rebuild required. Improvements
landed as additive rules in the Coder/Reporter/Validator agent prompts
plus a new `svg_docx.py` utility that the Reporter writes verbatim into
the Fargate executor sandbox at runtime (Step 0a pattern, since the
sandbox cannot import from the Runtime container).

## Fixes Shipped

### Text legibility (rcParams baseline)
- `text.color` / `axes.labelcolor` / `xtick.color` / `ytick.color`:
  `#000000` (was default gray)
- `axes.labelweight` / `axes.titleweight` / `figure.titleweight` /
  `font.weight`: `bold`

### Anti-clipping rules
- **Long y-label wrap**: 15+ char labels must split via `\n` or abbreviate
- **Combo chart ylim**: `top = max_value * 1.18` when value labels sit on bars
- **Inline label xlim**: explicit `xlim(0, max * 1.25)` for horizontal bars
  with long inline annotations
- **Reference line labels**: must use `ax.text()` with `transAxes` —
  `fig.text()` + `bbox_inches='tight'` causes axes to be squeezed into
  one figure corner

### Layout rules
- **Multi-panel grid avoidance**: prefer single panel + twin axes over
  2×1 grids when each subplot has 12+ x-ticks
- **Bubble cluster dedupe**: top 8 labels + radius 5% dedupe +
  adjustText fallback
- **Endpoint label clipping**: pad xlim past last label

### Data correctness
- **pandas index-alignment trap**: warn against
  `pd.DataFrame({col: series}, index=labels)` reindexing to NaN
- **Reference value labels**: don't repeat the reference value at every
  bar/point — annotate the line ONCE
- **calc_metadata merge**: read existing → merge by id → write (was
  overwriting each step)
- **Validator Chart Data Sanity Check**: catch all-zero / all-NaN
  primary value columns before reporting

### Architecture
- **SVG dual-embed (vector)**: `svg_docx.py` finalizes DOCX by injecting
  the OOXML `asvg:svgBlip` extension on every embedded PNG that has a
  matching `.svg` sibling — Word renders vector when available
- **Step 0a inline pattern**: Reporter prompt instructs the LLM to write
  `svg_docx.py` verbatim to `./artifacts/code/` before final DOCX save,
  bypassing Fargate sandbox import limits
- **MANDATORY POST-PROCESSING block** with `DO NOT REMOVE` marker:
  resists LLM mutation of the final SVG-embed step
- **Domain generalization**: prompt examples use `metric A` / `label_col`
  placeholders, not domain-specific terms

## Verification

8 production runs across 2 datasets, 14 charts per run. Every fix was
verified visually (chart-by-chart review) and programmatically (svgBlip
count, Caption style count, calc_metadata entry count).
